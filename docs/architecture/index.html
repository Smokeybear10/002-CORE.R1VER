§ 01 ♣

At a glance

The one-screen summary — what R1VER is, the key numbers, and the toggles that matter.

R1VER is a game-theoretic poker solver. It collapses the ~10160-node NLHE game tree by clustering ~3.1T strategically-equivalent hand+board situations into 542 buckets using Earth Mover's Distance over equity distributions, then runs external-sampling MCCFR self-play to produce a blueprint Nash strategy. The result is queryable in real time through an Actix Web API and an interactive Next.js explorer.

§ 04 ♠

The wire formats

Six tables. All keys are i64 — bit-packed obs and abs values store street, hash, and index in one column.
isomorphism 138M rows · 5 indexes
obsBIGINT — observation, bit-packed (hole, board)
absBIGINT — cluster ID this obs maps to
positionINTEGER — row_number within (PARTITION BY abs)
abstraction 542 rows · 4 indexes
absBIGINT — bit-packed cluster ID
streetSMALLINT — 0=P · 1=F · 2=T · 3=R
populationINTEGER — # isomorphisms in this cluster
equityREAL — expected win probability
centralityREAL — mean distance to other clusters
street 4 rows · 1 index
streetSMALLINT
nobsINTEGER — total isomorphisms on this street
nabsINTEGER — total abstractions on this street
metric 32k rows · 2 indexes
xorBIGINT — abs1 # abs2 (XOR of two cluster IDs)
dxREAL — Earth Mover's Distance between cluster pair
transitions 19k rows · 5 indexes
prevBIGINT — abstraction at street t
nextBIGINT — abstraction at street t+1
dxREAL — P(next | prev), Markov transition probability
blueprint — · MISSING
pastBIGINT — bucketed betting history
presentBIGINT — current cluster (the abs)
futureBIGINT — hashed action set available
edgeBIGINT — specific action (Fold/Call/Raise/...)
policyREAL — P(edge | infoset) at convergence
regretREAL — cumulative regret for this (infoset, edge)
i64 bit layout (obs / abs)
H · 8 bitsstreet index (0..3)
M · 44 bitsFNV-style hash of (street, index)
L · 12 bitscluster index 0..k
recover street(abs >> 56) as i16
recover index(abs & 0xFFF) as usize
